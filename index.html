Dropdown Menu Case Study

This is a case study on dropdown menus across various applications including: Hollister, AirBnB, and Google Slides. The purpose of this study is to anaylyze the differences in how these applications were implemented and catered to user needs/input. We will review how users can interact with dropdowns for each interface and how dropdowns as a feature can be improved upon. This study will also feature a revised design for how to make AirBnB's dropdowns more accessible.

Reflections

The designs for dropdown menus varied across platforms and input to cater to different user needs which allowed for unique and inclusive experiences. I particularly liked how the outputs for most of the designs were stylized to indicate to the user their current choices. This was the key focus of my redesign.

My design focused more on intuitive learnability and increased accessibility for keyboard users.

My design solves for a mismatch by providing more options for users with limited mobility to navigate dropdowns more easily. For example, in addition to being used for scrolling, holding the UP or DOWN arrow can be used to quickly and smoothly traverse through options as opposed to the user having to repreatedly TAB all the way through.

Google Slides is particularly difficult to navigate as a non-mouse user. The focus order is quite confusing as it jumps all over the page in counter-clockwise order as opposed to a more intuitive left-to-right, top-to-bottom. The dropdown options are also inaccessible to keyboard only users as it requires that users click on dropdowns to activate them. Hollister's dropdown menus use a black underline to indicate selection, but considering how much text appears on the dropdown menu, it can be hard to pinpoint which option is being selected. The same can be said about the outlined border since it is light and faint.

It is in my belief that mouse/touchpad users are most prioritized in the process of creating components as most of the population will default to the use of that input method. This can negatively impact those who rely more heavily on alternatives such as keyboard shortcuts or screenreaders because tools they also need to use may not be engineered in the most efficient or useful way for them.
